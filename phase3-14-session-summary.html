/session-summary

Lesson 14 of 15 Advanced 20 min

/today opens your day. /session-summary closes it. It reads today’s log entries, MEMORY.md, and current task state to produce an end-of-session report: what you completed, what was triaged, decisions made, and what to focus on tomorrow.

Prerequisites:

  • Completed Lessons 8, 10, 13 (/today, compound-reminder hook, side-gigs integration)
  • At least one task completed with /done today (a log entry exists)
  • MEMORY.md updated at least once

The Daily Workflow Loop

Morning: tasks → /today → see briefing During: /next → work → /done → /triage (as needed) Evening: /session-summary → update MEMORY.md → done

/session-summary is the bookend. It captures what happened so that tomorrow’s /today briefing has accurate data and your MEMORY.md has context for the next session.

Like /today, This Skill Forks

context: fork agent: Explore allowed-tools: Read, Glob, Grep

The summary involves reading multiple files and producing a formatted report. Forking keeps the intermediate work out of your main conversation context.

Walkthrough

1 Ensure session data exists

Check for today’s log file and MEMORY.md:

$ cat ~/work/_tasks/log/$(date +%Y-%m-%d).md $ cat ~/MEMORY.md

If either is missing, complete a task with /done and add some notes to MEMORY.md.

2 Install the skill
$ mkdir -p ~/.claude/skills/session-summary $ cp courses/phase3/lesson-14-session-summary/skill/session-summary/SKILL.md ~/.claude/skills/session-summary/SKILL.md
3 Run the summary
/session-summary

The Explore agent produces a report covering: tasks completed, tasks triaged, decisions from MEMORY.md, current state counts, and priority suggestions for tomorrow.

4 Verify accuracy

Cross-reference: Completed matches the log file, Current state matches ls ~/work/_tasks/pending/ | wc -l, Tomorrow suggestions are actionable and data-based.

Verification Checklist

  • /session-summary produces a multi-section end-of-session report
  • Completed tasks section matches today’s log entries
  • Current state counts are accurate
  • “Notes for Tomorrow” section provides actionable suggestions
  • Handles missing log file or MEMORY.md gracefully
  • Runs in a forked Explore agent (clean context)

Design Rationale

Why inject MEMORY.md via !`command`? MEMORY.md is usually short and is critical context for the “Decisions Made” section. Injecting it ensures the agent always sees it. Task files in pending are left for the agent to read via tools because there could be dozens.

Why not combine /today and /session-summary? They serve different purposes. /today is forward-looking (what should I do?). /session-summary is backward-looking (what did I do?). Separate skills keep each one concise.
