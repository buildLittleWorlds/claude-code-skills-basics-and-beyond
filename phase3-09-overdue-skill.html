/overdue Escalation Rules

Lesson 9 of 15 Intermediate–Advanced 25 min

/overdue implements a 4-tier escalation model: the longer you ignore a task, the louder the system gets. Tier 3+ tasks trigger display-popup overlays that demand a decision — the first skill where tmux and Claude interact during execution.

Prerequisites:

  • Completed Lessons 4–8 (all skills + startup script)
  • Completed Phase 2 Course 3 (reference files)
  • Pending tasks with at least 1–2 past-due dates

The 4-Tier Escalation Model

TierDays OverdueResponse
11–3Gentle reminder (listed in report)
24–7Highlighted with suggestion to complete or reschedule
38–14display-popup demanding a decision
415+Modal popup — blocks workflow until acknowledged

Key insight: The escalation is time-based, not priority-based. A P4 task 20 days overdue gets the same Tier 4 treatment as a P1 task. Chronic deferral is the problem being addressed, not task importance. If a task isn’t important enough to do, it should be dropped — not left in pending forever.

Walkthrough

1 Read the skill and reference
$ cat courses/phase3/lesson-09-overdue-skill/skill/overdue/SKILL.md $ cat courses/phase3/lesson-09-overdue-skill/skill/overdue/references/escalation-rules.md
2 Install the skill
$ mkdir -p ~/.claude/skills/overdue/references $ cp courses/phase3/lesson-09-overdue-skill/skill/overdue/SKILL.md ~/.claude/skills/overdue/SKILL.md $ cp courses/phase3/lesson-09-overdue-skill/skill/overdue/references/escalation-rules.md ~/.claude/skills/overdue/references/
3 Run the overdue check
/overdue

Claude reads pending tasks, calculates days overdue, groups by tier, displays the report, and shows a display-popup for any Tier 3+ tasks. Press q to dismiss the popup.

4 Try a modal popup manually
$ tmux display-popup -w 60 -h 10 "echo 'DECISION REQUIRED'; echo ''; echo 'Task: Fix API timeout'; echo 'Overdue: 7 days (Tier 3)'; echo ''; echo 'Press q to dismiss'; read"

This popup blocks input until dismissed — that’s the point for Tier 3+ tasks.

Verification Checklist

  • /overdue displays a tiered report of overdue tasks
  • Tasks are correctly assigned to tiers based on days overdue
  • Tier 3+ tasks trigger a display-popup with decision options
  • The popup is modal (doesn’t auto-close)
  • No files are modified (read-only)
  • “No overdue tasks” message appears when all tasks are current

Design Rationale

Why reference files? The escalation model is a tunable policy. Maybe Tier 3 should start at 10 days. With rules in a reference file, you edit one file and behavior changes.

Why not auto-reschedule? Automation should inform and prompt, not decide. A 20-day-overdue task might be worth doing or worth dropping. Only you know.

Phase 2 Callback: Course 3 (references/escalation-rules.md), Course 5 (!`command` preprocessing). Lesson 5 introduced display-popup as a 3-second confirmation; /overdue uses it as a modal for forced engagement.
