Compound Reminder Hook

Lesson 10 of 15 Intermediate–Advanced 25 min

Skills are instructions Claude follows when it chooses to. Hooks are rules that execute automatically. This Stop event hook reminds you to update MEMORY.md after completing tasks — ensuring tomorrow’s session has context.

Prerequisites:

  • Completed Lessons 4–9 (all skills working)
  • Completed Phase 2 Course 7 (hooks lifecycle, exit codes, JSON I/O, jq)
  • jq installed (brew install jq or apt-get install jq)

Concepts

The Infinite Loop Trap

When a Stop hook exits with code 2 (block with feedback), Claude responds to the feedback. But responding triggers another Stop event, which fires the hook again, which exits 2 again…

The solution is the stop_hook_active environment variable:

if [ "${stop_hook_active}" = "1" ]; then exit 0 fi export stop_hook_active=1

This guard ensures the hook only fires once per cycle. Without it, you get an infinite loop.

Hook Logic

  1. Guard: Check stop_hook_active — exit immediately if already fired
  2. Archive check: Use find -mmin -10 to detect recently completed tasks
  3. MEMORY.md check: Was ~/MEMORY.md updated in the last 10 minutes?
  4. Decision: If archives changed but MEMORY.md didn’t → exit 2 with reminder on stderr

Walkthrough

1 Read and install the hook
$ cat courses/phase3/lesson-10-compound-reminder/hooks/compound-reminder.sh $ mkdir -p ~/.claude/hooks $ cp courses/phase3/lesson-10-compound-reminder/hooks/compound-reminder.sh ~/.claude/hooks/ $ chmod +x ~/.claude/hooks/compound-reminder.sh
2 Register in settings

Add the hook to ~/.claude/settings.json:

{ "hooks": { "Stop": [ { "type": "command", "command": "bash ~/.claude/hooks/compound-reminder.sh" } ] } }
3 Test the reminder

Complete a task with /done, then ask Claude something simple. The hook fires and reminds you to update MEMORY.md.

4 Satisfy the hook
$ echo "## $(date +%Y-%m-%d) Session Notes" >> ~/MEMORY.md $ echo "- Completed: Plan weekend hike" >> ~/MEMORY.md

Now the next Stop event will find MEMORY.md recently updated and exit 0 (no reminder).

Verification Checklist

  • The hook fires after Claude responds (Stop event)
  • When a task was recently completed but MEMORY.md wasn’t updated, the reminder appears
  • When MEMORY.md was updated recently, no reminder appears
  • When no tasks were completed recently, no reminder appears
  • The stop_hook_active guard prevents infinite loops
  • The hook is registered in ~/.claude/settings.json

Design Rationale

Why a 10-minute window? The find -mmin -10 check creates a sliding window. If you completed a task 15 minutes ago, the hook stops reminding — you probably moved on.

Why stderr for feedback? When a hook exits with code 2, Claude reads stderr as the feedback message. Stdout is ignored for hook feedback.

Phase 2 Callback: Course 7 (Hooks): Stop event, exit codes (0=allow, 2=block+feedback), stderr for feedback messages, the stop_hook_active guard pattern.
