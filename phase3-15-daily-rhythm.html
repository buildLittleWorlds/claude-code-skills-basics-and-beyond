The Full Daily Rhythm

Lesson 15 of 15 Advanced 30 min

No new concepts, no new artifacts. This is the integration test — the complete daily rhythm, end-to-end, verified by a test script. Every skill, hook, and tmux pattern from the entire curriculum comes together here.

Prerequisites:

  • Completed Lessons 1–14 (all skills, hooks, cockpit, side-gigs integration)
  • All 6 skills installed: /next, /done, /triage, /today, /overdue, /session-summary
  • Both hooks installed: compound-reminder, file-protection
  • Cockpit startup script at ~/bin/tasks

The Daily Rhythm

┌─────────────────────────────────────────────┐ │ THE DAILY RHYTHM │ │ │ │ Morning: │ │ $ tasks ← launch │ │ → /today auto-runs ← briefing │ │ │ │ Work session: │ │ /triage ← process inbox│ │ /next ← pick task │ │ (do the work) ← actual work │ │ /done <task> <summary> ← complete │ │ (repeat) │ │ │ │ Periodic: │ │ /overdue ← check health │ │ │ │ End of day: │ │ Update ~/MEMORY.md ← reflect │ │ /session-summary ← wrap up │ │ Ctrl+J d (detach) ← leave │ │ │ │ Hooks (automatic): │ │ compound-reminder ← nag about MEMORY.md │ │ protect-task-files ← guard log/archive │ └─────────────────────────────────────────────┘

Walkthrough

1 Run the pre-flight check
$ bash courses/phase3/lesson-15-daily-rhythm/daily-rhythm-test.sh

Phase 1 of the script verifies all directories, all 6 skills, both hooks, and the startup script are in place. Fix any failures before proceeding.

2 Launch the cockpit

From a terminal outside tmux:

$ tasks

Wait for Claude to initialize and the /today briefing to appear automatically.

3 Run the full cycle
  1. /triage — Process the test inbox item
  2. /next — See the updated pending list
  3. /overdue — Check for escalated tasks
  4. /done integration-test Completed the daily rhythm test — Watch for confirmation popup, archive move, log entry, and compound-reminder hook
  5. Update ~/MEMORY.md with session notes
  6. /session-summary — Generate the end-of-session report
4 Verify the results

Return to the terminal where the test script is waiting and press Enter. It verifies: test task removed from inbox, removed from pending, present in archive with status: done, log entry exists, and MEMORY.md was updated today.

Verification Checklist

  • Integration test script passes all pre-flight checks
  • Full cycle works: triage → next → (work) → done → summary
  • Watchers update in real-time as tasks move between directories
  • Status bar reflects current counts
  • Compound-reminder hook fires after completing a task
  • File-protection hook blocks unauthorized edits
  • Integration test script passes all verification checks

Congratulations! Phase 3 is complete. The entire daily rhythm works end-to-end.
