Layout Build Sequence

  1. Split horizontally (top/bottom, 70/30)
  2. Split top vertically (left/right, 60/40)
  3. Split bottom vertically (left/right, 50/50)

The order matters because pane numbering shifts with each split. The script accounts for this.

Walkthrough

1 Read the enhanced script
$ cat courses/phase3/lesson-12-cockpit-layout/bin/tasks-v2

Compare with the v1 script: 4-pane splits with percentage sizing, embedded watchers, custom status bar with #() shell commands for live counts.

2 Install and launch
$ cp courses/phase3/lesson-12-cockpit-layout/bin/tasks-v2 ~/bin/tasks $ chmod +x ~/bin/tasks $ tmux kill-session -t tasks 2>/dev/null $ tasks

Verification Checklist

  • tasks launches a 4-pane HQ layout (Claude, shell, inbox watcher, pending watcher)
  • The status bar shows inbox count, pending count, overdue count, and time
  • Watchers refresh every 5 seconds
  • Status bar counts update every 10 seconds
  • Ctrl+J z zooms a pane to full screen and back
  • Adding/removing files is reflected in watchers and status bar
  • Claude receives /today automatically after startup

Design Rationale

Why 70/30 vertical split? Claude’s output benefits from height. Watchers are small, heads-up displays that don’t need much space.

Why embed watchers instead of separate windows? Window switching breaks visual continuity. With embedded watchers, you always see your task state — no switching required. The Ctrl+J z zoom handles cases where you need full-screen for one pane.
