The Startup Script

Lesson 7 of 15 Intermediate 25 min

Replace 2–3 minutes of manual tmux setup with a single command: tasks. The startup script creates a tmux session with Claude, watchers, and your skills pre-loaded — and auto-sends /today for your morning briefing.

Prerequisites:

  • Completed Lessons 1–6 (all manual operations + /next, /done, /triage)
  • Completed Phase 1 Lesson 9 (headless sessions, detached session creation)
  • ~/work/_tasks/ populated with seed data

Script Architecture

tasks (startup script) │ ├── Check: session exists? → reattach ├── Check: task directories exist? → error if not │ ├── Create session "tasks" │ ├── Window 0 "hq": Claude (pane 0) + Shell (pane 1) │ ├── Window 1 "inbox": watch inbox directory │ └── Window 2 "pending": watch pending with metadata │ ├── Launch Claude in hq.0 ├── Send /today after 4-second delay └── Attach to session

Walkthrough

1 Read the startup script
$ cat courses/phase3/lesson-07-startup-script/bin/tasks

Walk through each section: reattach guard (has-session), directory check, session creation (new-session -d), pane splits, watcher windows, Claude launch, delayed /today.

2 Install the script
$ mkdir -p ~/bin $ cp courses/phase3/lesson-07-startup-script/bin/tasks ~/bin/tasks $ chmod +x ~/bin/tasks

Ensure ~/bin is in your PATH, then verify:

$ which tasks
3 Launch the cockpit

Make sure you’re not inside tmux, then:

$ tasks

You should see: 3 windows (hq, inbox, pending), Claude starting in the left pane, and /today auto-sent after ~4 seconds.

4 Test reattach behavior

Detach (Ctrl+J d) and run tasks again. It reattaches instead of creating a duplicate.

5 Test live watchers

Create a test inbox item from the shell pane. Switch to the inbox window (Ctrl+J 1) — the new file appears within 5 seconds.

Verification Checklist

  • tasks command launches a 3-window tmux session (hq, inbox, pending)
  • HQ window has Claude in pane 0 and shell in pane 1
  • Inbox and pending watchers refresh every 5 seconds
  • Running tasks when a session exists reattaches instead of duplicating
  • Claude receives /today automatically after startup
  • The watcher reflects file changes in near-real-time

Design Rationale

Why a 4-second delay for /today? Claude Code needs a few seconds to initialize. The background subshell (sleep 4 && tmux send-keys ...) handles this timing without blocking the script.

Why watch instead of fswatch? watch is universally available, requires zero dependencies, and does exactly what we need — periodic refresh. For a 5-second refresh on a local directory, it’s the right level of sophistication.
