/triage with Dynamic Context

Lesson 6 of 15 Intermediate 30 min

/triage processes your inbox: classifying each item with domain, priority, due date, and tags, then moving it to pending. It introduces dynamic context injection (!`command`) and reference files working together.

Prerequisites:

  • Completed Lessons 4–5 (/next and /done skills working)
  • Completed Phase 2 Course 5 (dynamic context injection with !`command`)
  • Completed Phase 2 Course 3 (reference files, references/ directory)
  • Inbox files in ~/work/_tasks/inbox/

Concepts

How Dynamic Context Works

In the SKILL.md, lines like !`ls ~/work/_tasks/inbox/` run as shell commands before Claude sees the prompt. The output replaces the placeholder. Claude receives the rendered result with actual file names — preprocessing, not execution.

Reference Files

The references/ directory contains routing-rules.md (domain classification) and tagging-guide.md (tag taxonomy). These encode business logic that should be consistent across invocations.

~/.claude/skills/triage/ ├── SKILL.md └── references/ ├── routing-rules.md └── tagging-guide.md

Walkthrough

1 Examine the skill and references
$ cat courses/phase3/lesson-06-triage-skill/skill/triage/SKILL.md $ cat courses/phase3/lesson-06-triage-skill/skill/triage/references/routing-rules.md $ cat courses/phase3/lesson-06-triage-skill/skill/triage/references/tagging-guide.md
2 Install the skill with references
$ mkdir -p ~/.claude/skills/triage/references $ cp courses/phase3/lesson-06-triage-skill/skill/triage/SKILL.md ~/.claude/skills/triage/SKILL.md $ cp courses/phase3/lesson-06-triage-skill/skill/triage/references/routing-rules.md ~/.claude/skills/triage/references/ $ cp courses/phase3/lesson-06-triage-skill/skill/triage/references/tagging-guide.md ~/.claude/skills/triage/references/
3 Run triage
/triage

Claude reads each inbox file, applies routing rules, assigns metadata, moves files to pending, and displays a summary.

4 Verify results
$ ls ~/work/_tasks/inbox/ # Should be empty $ ls ~/work/_tasks/pending/ # Should have new files $ cat ~/work/_tasks/pending/research-state-management.md # Check metadata

Each triaged file should now have domain, priority, due, and tags fields filled in.

Verification Checklist

  • /triage processes all inbox files in a single pass
  • Each triaged file has domain, priority, due, and tags assigned
  • Files are moved from inbox/ to pending/
  • The triage summary shows counts by domain and priority
  • Running /triage on an empty inbox produces a helpful message
  • Routing decisions are consistent with routing-rules.md
  • Tags come from the taxonomy in tagging-guide.md
  • /next shows the newly triaged tasks

Design Rationale

Why dynamic context instead of letting Claude run ls? The !`command` preprocessing is faster (runs before context loads) and more reliable (no Bash tool access needed).

Why reference files instead of inline rules? Reference files are maintainable. Add a new domain or tag by editing one file instead of rewriting SKILL.md.

Why not context: fork? Triage modifies files. A forked Explore agent is read-only. And the summary table is useful in your main conversation for follow-up questions.

Phase 2 Callback: Course 5 (!`command` dynamic context), Course 3 (references/ directory), Course 6 (sequential workflow with dependencies).
