/today Morning Briefing

Lesson 8 of 15 Intermediate–Advanced 25 min

When you launch the cockpit, the first thing you want is a snapshot of your day. /today produces a morning briefing — overdue items, due today, coming this week, recent completions, and suggested focus. It’s the first skill to use forked subagent execution.

Prerequisites:

  • Completed Lesson 7 (startup script, cockpit running)
  • Completed Phase 2 Course 5 (context: fork, agent: Explore, allowed-tools)
  • Pending tasks and log files in ~/work/_tasks/

The Fork Tradeoff

AspectInline (/next)Forked (/today)
Context costAll intermediate steps visibleOnly final result returned
SpeedFaster (no agent startup)Slightly slower (agent init)
Follow-upCan ask “tell me more about task 2”Agent is gone — can’t follow up
Tool accessFull tool setOnly allowed-tools
Best forSmall queries, interactive workReports, analysis, read-heavy tasks

Rule of thumb: Fork when the task is self-contained and its intermediate output would clutter your main conversation.

Agent and Tool Choices

context: fork agent: Explore allowed-tools: Read, Glob, Grep

Explore agent uses Haiku (fast, cheap). The briefing is read-only. Allowed-tools explicitly restricts to read-only tools.

Walkthrough

1 Read and install
$ cat courses/phase3/lesson-08-today-skill/skill/today/SKILL.md $ mkdir -p ~/.claude/skills/today $ cp courses/phase3/lesson-08-today-skill/skill/today/SKILL.md ~/.claude/skills/today/SKILL.md
2 Test the briefing
/today

The Explore agent reads task files and returns a formatted briefing with five sections: overdue, due today, this week, recent completions, and suggested focus.

3 Test the cockpit auto-briefing

Kill the session and relaunch:

$ tmux kill-session -t tasks $ tasks

After Claude initializes (~4 seconds), the startup script sends /today automatically. This is the daily experience: type tasks, get your briefing, start working.

Verification Checklist

  • /today produces a multi-section morning briefing
  • Runs in a forked Explore agent (brief pause, clean output)
  • Overdue, due-today, and this-week sections are accurate against your data
  • Recent completions pulls from log files
  • Suggested focus prioritizes overdue and high-priority items
  • The cockpit startup script auto-sends /today

Design Rationale

Why Explore instead of general-purpose? Haiku is faster and cheaper. The briefing is pure analysis: read files, compare dates, format output. No modifications needed.

Why dynamic context for counts but not file contents? Counts are one-line values injected via !`command`. Actual file contents are left for the agent to read — injecting all files would bloat the prompt.

Phase 2 Callback: Course 5 (context: fork + agent: Explore + allowed-tools). Same pattern as the investigate-issue example, applied to task data.

What’s Next: /today shows what’s overdue, but treats all overdue tasks equally. Lesson 9: /overdue Skill adds a 4-tier escalation model with increasingly urgent treatment based on how far past due a task is.
