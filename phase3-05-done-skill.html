/done with Arguments

Lesson 5 of 15 Intermediate 30 min

/done closes the task loop: it marks a task as completed, moves it to the archive, logs the completion, and shows a floating popup. This skill introduces $ARGUMENTS for parameterized input and display-popup for tmux overlay confirmations.

Prerequisites:

  • Completed Lesson 4 (/next skill installed and working)
  • Completed Phase 2 Course 5 (arguments, $ARGUMENTS, argument-hint)
  • Completed Phase 2 Course 6 (multi-step workflows, validation gates)

New Concepts

Comparing /next and /done

Feature/next/done
ArgumentsNone$0 keyword + remaining text as summary
argument-hintNone[task-keyword] [completion summary]
Workflow steps1 (read + display)7 (find → confirm → update → move → log → popup → suggest)
File modificationsNoneMoves file, edits frontmatter, appends to log
User confirmationNoneExplicit y/n before changes

The display-popup Command

$ tmux display-popup -d <seconds> -w <width> -h <height> "<command>"

The popup floats in the center of the current pane. With -d 3, it auto-closes after 3 seconds. This is the same mechanism that /overdue (Lesson 9) uses for its Tier 3 escalation popup.

Walkthrough

1 Read and install the skill
$ cat courses/phase3/lesson-05-done-skill/skill/done/SKILL.md $ mkdir -p ~/.claude/skills/done $ cp courses/phase3/lesson-05-done-skill/skill/done/SKILL.md ~/.claude/skills/done/SKILL.md
2 Complete a task with full arguments
/done fix-api-timeout Reduced default page size to 100 and added cursor-based pagination

Claude finds the task, shows a confirmation, waits for your “yes,” then moves the file, creates a log entry, and shows a floating popup.

3 Validate the completion
$ bash courses/phase3/lesson-05-done-skill/skill/done/scripts/validate-done.sh fix-api-timeout.md

You should see 6 green checkmarks verifying the file move, archive status, and log entry.

4 Test edge cases

Test without a summary (/done wiki — Claude asks for one). Test with a non-matching keyword (/done nonexistent-task — helpful error). Test with an ambiguous keyword (/done plan — multiple matches listed).

5 Try display-popup manually
$ tmux display-popup -d 3 -w 40 -h 5 "echo '✓ Task completed!'"

A floating box appears for 3 seconds and vanishes. This is what the skill triggers.

Verification Checklist

  • /done fix-api-timeout <summary> finds the task, confirms, moves to archive, and logs
  • validate-done.sh passes all 6 checks
  • /done wiki (no summary) prompts for a completion summary
  • /done nonexistent returns a helpful “not found” message
  • /next shows the updated list with completed tasks removed
  • tmux display-popup shows a floating overlay that auto-closes
  • Today’s log file contains entries for each completed task

Design Rationale

Why require confirmation? Destructive-ish operations (moving files, appending logs) should have a gate. If the keyword matched the wrong task, you want to catch it before the move happens. This is the “validation gate” pattern from Phase 2 Course 6.

Why append-only logs? The log directory is an audit trail. You never edit past entries. The /session-summary skill (Lesson 14) reads these logs to produce end-of-day reports.

Phase 2 Callback: Course 5 ($ARGUMENTS, argument-hint), Course 6 (sequential workflows with validation gates), Course 3 (scripts/validate-done.sh in the skill’s scripts/ directory).
