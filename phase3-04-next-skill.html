First Skill — /next

Lesson 4 of 15 Intermediate 25 min

Replace the fragile bash one-liner from Lesson 3 with /next — a proper Claude Code skill that reads your pending tasks, sorts them by due date, highlights overdue items, and displays a clean formatted table. One slash command instead of a brittle script.

Prerequisites:

  • Completed Lessons 1–3 (send-keys, capture-pane, multi-session)
  • Completed Phase 2 Courses 1–3 (skill anatomy, descriptions, full skill structure)
  • Seed data in ~/work/_tasks/

Design Decisions

Before building, think through the choices. Each maps to Phase 2 concepts:

DecisionChoiceWhy
disable-model-invocationtrueQuery you run deliberately — no auto-triggering
context: forkNot usedOutput is small; you want it in main context for follow-up
allowed-toolsDefault setNeeds Read, Glob, Grep. Instructions say “do not modify files”
Shell script?NoClaude handles YAML edge cases that break grep/sed

Walkthrough

1 Read the SKILL.md
$ cat courses/phase3/lesson-04-next-skill/skill/next/SKILL.md

Notice the structure: frontmatter with name, description, disable-model-invocation: true. Numbered steps (read → parse → sort → display). Explicit output format template. Read-only rules.

2 Install the skill
$ mkdir -p ~/.claude/skills/next $ cp courses/phase3/lesson-04-next-skill/skill/next/SKILL.md ~/.claude/skills/next/SKILL.md
3 Test with direct invocation

In Claude Code:

/next

You should see overdue tasks marked with a warning indicator, tasks sorted earliest-first, and a summary line at the bottom.

4 Test the empty state

Temporarily move all pending files to see the empty-state behavior:

$ mkdir -p /tmp/tasks-backup $ mv ~/work/_tasks/pending/*.md /tmp/tasks-backup/

Run /next again. Claude should display the “No pending tasks” message. Restore:

$ mv /tmp/tasks-backup/*.md ~/work/_tasks/pending/ $ rmdir /tmp/tasks-backup

Verification Checklist

  • /next displays a sorted table of pending tasks from ~/work/_tasks/pending/
  • Overdue tasks (due date before today) are visually distinguished
  • The summary line shows total count, overdue count, and due-today count
  • An empty pending directory produces a helpful message
  • The skill appears in /help output

Design Rationale

Why is this the first skill? It’s the simplest useful skill for the cockpit: read-only, no arguments, no dynamic context, no forking. Each subsequent skill adds one more feature — /done adds arguments, /triage adds dynamic context, /today adds forking. The complexity ramp is deliberate.

Why sort by due date instead of priority? Due date creates urgency-first ordering. A P3 task due yesterday is more urgent than a P1 task due next month. Priority is a secondary signal shown for context, not the sort key.

Phase 2 Callback: This builds on Course 1 (Your First Skill), Course 2 (Descriptions and Triggers), and Course 3 (Full Skill Anatomy). Same SKILL.md structure as daily-standup, with disable-model-invocation to prevent auto-triggering.

What’s Next: /next shows you what to work on. Lesson 5: /done Skill closes the loop — completing a task with $ARGUMENTS, moving it to archive, logging the completion, and showing a floating display-popup confirmation.
